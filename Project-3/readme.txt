When deployed the link to the WADL should be - http://localhost:8080/AppointmentManager/webresources/application.wadl

The Client has been tested from within Internet Explorer - with the Access Data Sources across domains property set to Enabled - to enable CORS
Setting available under Tools > InternetOptions > Security Tab > Miscellaneous > Access Data Sources across domains - Enabled

The Client allows for only one labTest to be updated on an existing appointment as well as added to a newly created appointment
